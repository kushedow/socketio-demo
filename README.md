## Функциональность
Сервер поддерживает следующие возможности через Socket.IO:
- Подтверждение подключения (`welcome` сообщение).
- Валидация входящих JSON-данных.
- Эхо-события и обработка простых текстовых команд.
- Разделение логики инициализации приложения и регистрации событий.

## Требования
- Python 3.12+
- Зависимости из `requirements.txt`

## Установка

1. Клонируйте репозиторий (или скачайте файлы).
2. Создайте виртуальное окружение:
   ```bash
   python -m venv venv
   ```
3. Активируйте виртуальное окружение:
   - macOS/Linux: `source venv/bin/activate`
   - Windows: `venv\Scripts\activate`
4. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## Запуск

Для запуска сервера используйте команду:
```bash
python main.py
```
Сервер будет доступен по адресу `http://127.0.0.1:5000`.

## События Socket.IO

### Исходящие события (от сервера к клиенту)
- `welcome`: Отправляется через 0.5с после подключения. Данные: `{"code": 7007}`.
- `status_update`: Ответ на событие `join`. Данные: `{"status": "joined", "code": 8888}`.
- `error`: Отправляется при ошибке валидации или неверном формате сообщения.

### Входящие события (от клиента к серверу)
- `connect`: Автоматическое событие при подключении.
- `join`: Ожидает JSON-объект. Возвращает `status_update`.
- `hello`: Принимает JSON и возвращает его же клиенту в событии `hello`.
- `echo`: Принимает JSON и возвращает его же клиенту в событии `echo`.
- `message`: Ожидает объект вида `{"body": "Привет!"}`.
  - Если body == "Привет!", сервер ответит `{"message": "Привет от сервера!", "code": 1234}`.
  - В противном случае вернет ошибку.

## Структура проекта
- `main.py` — точка входа, запуск сервера uvicorn.
- `components.py` — инициализация FastAPI и Socket.IO сервера.
- `src/events.py` — логика обработки событий Socket.IO.
- `requirements.txt` — список зависимостей проекта.
- `ruff.toml` — конфигурация линтера и форматировщика Ruff.
